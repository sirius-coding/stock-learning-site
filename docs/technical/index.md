# 技术分析导读

## 先说明边界

技术分析不是预测未来的按钮，更不是脱离企业质量、估值和风险管理单独工作的系统。它更适合回答三个问题：

- 现在是强势、弱势还是震荡
- 这个位置是追高、回踩还是破位
- 什么时候更适合观察，什么时候更适合执行

## 这组内容怎么读

技术分析更适合用来辅助观察趋势、位置和执行，而不是替代企业分析和估值框架。建议阅读顺序是：

1. `K线`
2. `成交量`
3. `均线`
4. `MACD`
5. `布林线`
6. `分时`
7. `筹码`
8. `盘口`

## 为什么这样排

- 先看 `K线`，因为它是最直观的价格语言。
- 再看 `成交量`，因为量价要配合。
- `均线 / MACD / 布林线` 属于趋势和节奏工具。
- `分时 / 筹码 / 盘口` 更偏执行层观察。

## 使用原则

- 不神化单一指标
- 大周期优先于小周期
- 先有逻辑，再用技术图形找位置
- 技术分析用于辅助，不用于代替风险管理

## 建议入口

- 新手先读 [K线基础版](./k-line-basic)。
- 想补量价关系，接着读 [成交量基础版](./volume-basic)。
